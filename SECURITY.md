# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in any Groundlens project, please report it
**privately**. Do **not** open a public issue, pull request, or discussion for security
matters.

Email <javier@groundlens.dev> with:

- a description of the vulnerability and its impact,
- the affected project and version,
- steps to reproduce (a minimal proof of concept if possible).

You can expect an acknowledgement within **5 business days**. We will work with you to
understand and resolve the issue, and will credit reporters who wish to be named once a
fix is released.

## Supported versions

Groundlens projects are in active development. Security fixes are applied to the latest
released version of each project on the default branch. Please make sure you are running
the most recent version before reporting.

## Scope

This policy covers the source code in the [groundlens-dev](https://github.com/groundlens-dev)
organization. Vulnerabilities in third-party dependencies should be reported to their
respective maintainers; we are happy to help coordinate where a Groundlens project is
affected.
