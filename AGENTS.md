# AGENTS.md

Guidance for AI agents (and humans) working in this repository.

## What this repository is

This is the **Groundlens GitHub profile repository** (`groundlens-dev/groundlens-dev`). It
renders the profile landing page and holds the community health files. It contains:

- `README.md` — the profile landing page (rendered at the top of the org/user profile)
- `assets/` — images used by the README (hero, logos, screenshots)
- community files — `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md`, `SECURITY.md`, `LICENSE`

It is not a code project; there is no build or test step. The deliverable is the rendered
README and the community files.

## Voice and style (non-negotiable)

- **No emojis anywhere.** Use clean text headings; for iconography use SVG assets, never
  emoji. Shields.io badges with brand logos (`logo=`) are acceptable.
- **Sell the project, not the person.** The page presents Groundlens (the practice and its
  two lines, Groundlens and Otwin), not a personal CV. Keep any biographical content brief
  and at the end.
- **Measured and rigorous.** Authoritative, calm, engineering tone. Avoid hype and
  superlatives. State limitations plainly. Do not label work as "honest" — demonstrate it.
- **Brand line:** *Geometric methods for trustworthy models — verification over capability.*

## Conventions

- Section order in the README: Groundlens, Otwin, Featured, Research, community sections,
  License, About.
- Images in this repo are referenced with **relative paths** (`assets/...`). Images from
  the `otwin` repo are referenced with **raw URLs**
  (`https://raw.githubusercontent.com/groundlens-dev/otwin/main/...`).
- Before linking any image, confirm the file exists; a broken image is worse than no image.
- Keep links current; verify external URLs (arXiv, Hugging Face, LinkedIn) resolve.

## Do not

- Do not add emojis.
- Do not turn the page into a personal résumé.
- Do not introduce unverified claims, numbers, or links.
