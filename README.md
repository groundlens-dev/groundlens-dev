# Groundlens

### Geometric methods for trustworthy models

Verification over capability · deterministic · auditable · candid about limits

<br>

[![Website](https://img.shields.io/badge/groundlens.dev-fc7604?style=for-the-badge&logo=astro&logoColor=white)](https://groundlens.dev)

</div>

<br>

Groundlens applies **geometry** to make model outputs trustworthy — across two domains. Embedding geometry to verify *what an LLM says*; energy geometry to model *how a physical system behaves*. Both deterministic where possible, calibrated where not, and explicit about where the method stops working.

### Groundlens

**LLM output verification.** Geometric grounding / hallucination triage for production LLMs in regulated industries — deterministic scores, sub-second, **no second LLM in the loop**.

Three papers (Semantic Grounding Index · Geometric Taxonomy of Hallucinations · Rotational Dynamics).

### Otwin

**Physics-informed digital twins** with **calibrated uncertainty** for grid-scale energy storage and other physical systems. Port-Hamiltonian structure (energy geometry), leakage-free validation, CPU-first.

Spans white-box (pumped hydro, DC motor) to grey-box (battery State-of-Health).
